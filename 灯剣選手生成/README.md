# 灯剣選手生成

Excel マクロブック `灯剣選手生成完成版.xlsm` の主要機能を、ブラウザで動くローカル生成システムに移した試作版です。

## できること

- 国籍グループを選んで選手を一括生成
- 人数とシードを指定して、同じ条件の結果を再生成
- 氏名、性別、年齢、国籍、身長、利き手、能力上位、ポジション適性を表示
- 対象リーグごとの国籍比率を目安にした国籍抽選
- `全ステータス` をオンにすると、30項目すべての能力値を一覧表示
- 選手行を選ぶと、全能力値と全ポジション適性を詳細表示
- 複数ポジションを本職級にこなせる選手も生成
- チームプランナーで、生成済みメンバーから重複なしの編成案を作成
- 生成候補から選手を採用し、採用チームとして独立管理
- 採用選手の基本情報、ステータス、ポジション慣れを編集
- 手入力で選手を追加
- 採用チームをブラウザに保存し、JSONで出力
- 主ポジション、傾向、強さ、年齢帯、性別、利き手を指定した狙い生成
- 強度、人数、基本形、傾向、年齢構成を指定したチーム一括生成
- `㊙項目` をオンにすると、元ブックの隠しシート相当の追加列を表示
- 結果を JSON で保存

## 起動方法

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

起動後、ブラウザで `http://127.0.0.1:4173/` を開きます。`index.html` の直接オープンでも動くように `data/generator-data.js` を同梱していますが、安定運用ではローカルサーバー起動を推奨します。

## ファイル構成

- `index.html`: 画面
- `styles.css`: 見た目
- `app.js`: 生成ロジック
- `data/generator-data.json`: Excel から抽出したマスタデータ
- `data/generator-data.js`: file 直開き用のマスタデータ
- `scripts/extract_workbook_data.py`: 元ブックから JSON を再生成する抽出スクリプト
- `scripts/import_name_pools.js`: 追加の国別名前プール JSON を取り込むスクリプト
- `analyze_workbook.py`: 元ブックの構造確認用スクリプト

## 移植方針

元ブックは、`生成機` と `㊙` のシートで大量の乱数式を使い、`名前リスト`、`国籍リスト`、`国籍名前対照`、`特能` をマスタとして参照しています。

この版では、Excel の揮発関数に依存せず、シード付き乱数で生成します。完全に同じ乱数列を再現するものではありませんが、名前・国籍・能力名・ポジション重みなどのマスタは元ブックから抽出しています。

能力値は、単純に各項目を独立抽選するのではなく、先に役割タイプを決め、才能、年齢曲線、身体傾向、能力カテゴリ、役割ごとの重みを反映して生成します。これにより、守備型は守備系、展開型は視野やパス系がまとまりやすくなります。

生成時の役割タイプには `#20` を使いません。`#20` は互換用の適性項目として残しますが、新規生成選手の型には出さない方針です。

国籍は、世界ランキングメモと日本・ドイツの例を参考にしたリーグ別の重みで抽選します。現在の名前データベースに存在する国籍だけを生成候補に使うため、未収録の国は名前データ追加後に比率へ反映しやすい形で残しています。

追加の国別名前プールは `namePoolsByCountry` として保持し、サブグループがある国は同じサブグループ内で名前を組み合わせます。ミャンマーは継承姓なしの個人名、エチオピアは個人名＋父称要素、コンゴ民主は個人名＋姓/ポストノム要素のように、国ごとの命名構造を反映して表示名を組み立てます。

狙い生成では、主ポジション、選手傾向、強さ、年齢帯、性別、利き手を指定できます。国籍範囲は通常生成と同じく画面上部の「対象」を使います。未指定の項目はランダムに残るため、条件に寄せつつ個体差のある候補を生成できます。

チーム生成では、登録20人、ベンチ入り16人、スタメン7人を選び、強度ランク、強度補正、ばらつき、スター人数、控え層、基本フォーメーション、チーム傾向、年齢構成を指定できます。内部ではエース級、主力級、ローテ級、控え級、育成枠の層を作り、それぞれの目標適性帯へ寄せて選手を生成します。生成結果は候補一覧だけに出すことも、採用チームへ置換または追加することもできます。採用チームは登録上限の20人を超えないように制限します。

リーグ自国籍の選手は、チーム生成時に約8割を控え・育成・ローテ寄りの目標帯へ内部的に補正します。自国籍であること自体は自然な前提として扱い、画面上は通常の `主力級`、`ローテ級`、`控え級`、`育成枠` として表示します。残りの自国籍選手は通常どおり主力級やエース級にもなり得ます。

ポジション適性は、能力から選んだ主ポジションを倍率 `100%` として元の適性値をそのまま表示し、周辺から少し広めの候補へ `主級`、`副A`、`副B`、`副C`、`副D` の段階を個体差つきで付与します。副適性が付かない選手もいます。遠いポジションは低めの未習熟値になります。右利きは右側、左利きは左側の同サイドポジションに個体差つきのマイナス補正が入ります。

チームプランナーは、基本フォーメーション `3-1-3`、`2-2-3`、`2-3-2`、`3-3-1`、`2-4-1`、`3-2-2`、`4-2-1`、`4-1-2` を選び、採用チームの選手を重複なしで各スロットに割り当てます。各スロットには基本役割、置換可能役割、OP役割を候補として持たせ、その候補内で最も適性値が高い選手を優先します。

OPは置換可能指定として扱います。`#11` は `#1`、`#10/#14` は `#4`、`#15` は `#5`、`#16` は `#6`、`#17` は `#7` のOP置換候補です。各フォーメーションのOP一覧に含まれる場合だけ、そのスロットの候補に自動追加します。

ポジション体系は、`#11` を `Libero Guard` として `#1`、`#14`、`#4` に近い中央守備系で扱います。左右ペア補正は `#2/#3`、`#5/#6`、`#12/#13`、`#15/#16` に適用し、`#11` は左右サイド補正の対象にしません。
